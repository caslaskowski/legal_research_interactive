The Life of a Statute

The same statute lives in several places at once — as a slip law, as a session law bound in chronological order, and as a code section organized by subject. Knowing which version you are looking at, and which one to research in, is the chapter’s foundational takeaway. Walk the journey, then compare the two versions researchers actually choose between.

How to use this: click each stage of the journey, then compare the bare code against the annotated code, and take the two checks. The next module in this chapter puts the finding aids on top of this structure.

The comparison that decides where you work

Both contain the same statutory text. Only one connects that text to everything else.

The code (official text)

The statute’s current text, organized by subject into titles and sections — the words that govern.

  • The operative text itself
  • A history note tracing back to the public law

Use it for: the authoritative words. What it cannot do: tell you what courts have made of them.

The annotated code (U.S.C.A. / U.S.C.S.)

The same text, turned into a research hub by everything editors attach to it.

  • Notes of Decisions — the cases interpreting each section
  • Cross-references to regulations and related sections
  • History notes and secondary-source leads

Use it for: research. The annotations turn bare text into a hub that connects to the rest of the legal landscape.
